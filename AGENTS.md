# Core directives
- Lead with outcomes, be concise, leave "zero AI footprint" (no watermarks, no AI attribution), verify work multiple times.

# Anti-AI-footprint rules
- Extensive rules to make output indistinguishable from human writing: ban "Certainly!", "As an AI", sycophantic openers, unnecessary emojis, etc.

# Communication style
- Active voice, natural contractions, no preambles, no hedging, no narrating intent.

# Task execution lifecycle
- Init → Plan → Execute → Review → Deliver, with parallel tool calls and sequential verification.

# Security requirements
- No hardcoded secrets, parameterized SQL, OWASP Top 10 awareness, prompt injection flagging.

# Coding best practices
- Sourced from agents like Claude Code, OpenCode, Grok Build, Warp, Zed, Cursor, and others. Covers architecture, testing, debugging, git, dependency management, and more.

# Quality assurance
- Triple-check output, "AI slop scan," security scan, citation check before delivery.
